Exploring together

A technical white paper on the architecture of a team of autonomous agents that covers a map without losing touch — the models, the mathematics, and the experiments that reshaped them. Written for readers with an undergraduate CS background: it keeps the formalism and defines every symbol as it goes. Companion to the formal Paper, the Theory page (full formalism), and the Findings.

Thesis Across a multi-year search, the dominant lever was not the function approximator (perception network, memory, attention) but the reward decomposition. Replacing a shared team return with a per-agent difference reward — a counterfactual marginal-contribution signal — produced larger gains than any architectural change. The bottleneck was the credit-assignment signal and the action representation, not information.

01 · setting

Problem & formal setting

We study a team of N agents that act on local information, communicate only over a range-limited, time-varying graph, and are scored by a single team-level objective. The formal object is a finite-horizon cooperative Dec-POMDP — a decentralized, partially observable Markov decision process [1]. It is the multi-agent generalization of the MDP most CS students meet in an intro-RL course, with two twists: several agents act at once, and none of them sees the true state. The tuple is:

\langle\, I,\ S,\ \{A_i\},\ \{\Omega_i\},\ T,\ O,\ R,\ \rho_0,\ H \,\rangle
(1)

Every symbol, with its concrete meaning in our grid world:

The Dec-POMDP tuple, element by element
I = {1,…,N}
the set of agents.
S
the state — the complete, god's-eye configuration of the world at one step: every agent's grid cell, the set of cells already covered, the obstacle map, and the current communication graph. It is everything that is true. Crucially, no single agent ever observes S directly.
A_i
agent i's action set — here the five one-step moves {N, S, E, W, stay}. The joint action is a_t = (a_{1,t},…,a_{N,t}).
Ω_i
agent i's observation set — the space of local readings it can receive (see O).
T(s'|s,a)
the transition model — the environment's rulebook: given state s and joint action a, the probability of the next state s'. In our grid it is nearly deterministic — each move updates a position (a move into a wall is a no-op), the covered set grows, and the graph is recomputed — with randomness entering mainly through where things start (ρ₀).
O(o|s,i)
the observation model — this is where partial observability lives (unpacked below).
R(s,a)
the reward — one scalar handed to the whole team each step (e.g. count of newly covered cells, minus collision/connectivity penalties). Because it is a single number, it does not say which agent earned it — the credit-assignment problem of §4.2.
ρ₀
the initial-state distribution — how episodes start (random agent placements and obstacle layout).
H
the horizon — episode length (H = 100 steps here). "Finite-horizon" means the mission is judged over a fixed window, which is what creates the budget ceiling of §6.

Where partial observability enters

In an ordinary MDP the agent is assumed to see the full state s_t and choose its action from it. A POMDP breaks that assumption. The state s_t still evolves under T, but no agent observes it. Instead, after each transition the observation model O emits, for each agent i, a local observation drawn from Ω_i:

o_{i,t} \sim O(\,\cdot \mid s_t,\, i\,), \qquad o_{i,t} \in \Omega_i

Here O returns exactly what agent i can physically sense — the occupancy of the cells in a small window around it, its own coordinates, and the messages arriving from in-range neighbors — and nothing about distant regions or out-of-range teammates. Partial observability is precisely this gap between s_t (everything true) and o_{i,t} (the sliver agent i receives). It is why an agent cannot simply "look up" the global coverage or the connectivity, and why it must build up a belief over time (§4.3). Formally, a plain MDP is the special case o_{i,t} = s_t.

The objective: joint policy, expectation, trajectory

Each agent has its own decision rule π_i. Bundling them gives the joint policy π = (π_1,…,π_N). It is called joint because the mission outcome depends on all agents together — no agent's contribution is scored in isolation; they jointly determine what happens. Running π from a start state produces a trajectory — the actual sequence the episode traces out:

\tau = (\, s_0,\, a_0,\, s_1,\, a_1,\, \ldots,\, s_H \,)

Because the start state (ρ₀) is drawn at random and the policies are stochastic (they sample actions), τ is a random object: run the same π twice and you get two different trajectories. (In the general Dec-POMDP the transition T can add further randomness too; in our grid it is nearly deterministic, so ρ₀ and the policies are the operative sources.) The mission return is the total reward summed along one trajectory, and J(π) is its expected value — the probability-weighted average over all trajectories π could produce:

J(\pi) = \mathbb{E}_{\tau \sim \pi}\!\left[\, \sum_{t=0}^{H} R(s_t,\, a_t) \,\right]
(2)

where E_{τ~π}[·] is the expectation (average) over trajectories induced by π, and the sum runs over the H steps. Training seeks the π that maximizes J(π). What makes the multi-agent case distinctive is a decomposition into three layers, where interaction enters the mission twice — shaping each agent's input, and shaping how the joint action composes into the next state.

Micro — the individual agent (and: history vs. trajectory)

Agent i's local input at step t is x_{i,t} = (o_{i,t}, q_{i,t}): its observation o_{i,t} (what it senses, above) plus an auxiliary vector q_{i,t} holding the other things it receives this step — chiefly the aggregated neighbor messages (c_{i,t} below) and its role tag. Because one input is a thin slice of the world, the agent's decision depends on its whole history — the sequence of the full inputs it has taken in, interleaved with its own past actions:

h_{i,t} = (\, x_{i,0},\, a_{i,0},\, x_{i,1},\, a_{i,1},\, \ldots,\, x_{i,t} \,)

Note this accumulates the full inputs x — each already carrying the messages the agent received — not the bare observations o. (The textbook "action–observation history" writes o only because there the observation is defined to subsume received communication; since we split messages out into q, the history must carry x.) The belief b_{i,t} — the agent's occupancy map, its connectivity estimate λ̂₂, and trust features — is a learned sufficient statistic of this history, so in practice the policy conditions on the belief rather than replaying the whole sequence: a_{i,t} ~ π_i(·|h_{i,t}) ≈ π_i(·|b_{i,t}).

History vs. trajectory — don't conflate them The trajectory τ is the global, god's-eye record of the whole episode — all states and joint actions. The history h_{i,t} is one agent's local, partial record — only its own past inputs and actions. An agent never sees τ; it only ever has h_{i,t}. That asymmetry is the whole difficulty.

Bridge — the interaction layer

A time-varying directed graph G_t = (I, E_t) defines who can talk to whom at step t. Agent i's in-range neighbors are N_{i,t}. It receives a message m_{j→i,t} from each neighbor j and must fuse a variable number of them into a fixed-size context c_{i,t} via an aggregation operator:

c_{i,t} = \mathrm{Agg}_i\!\big(\, \{\, m_{j\to i,\,t} \;:\; j \in \mathcal{N}_{i,t} \,\} \,\big)
(3)
What Agg has to be (first look; details in §4) The neighbor set changes size step to step and has no natural ordering, so Agg_i must accept a variable count of messages and be permutation-invariant — the result can't depend on the order the neighbors happen to be listed. That restricts it to symmetric pooling: a sum, mean, max, or an attention-weighted average over the messages. It is the only channel through which information crosses between agents, which is why §4 treats the choice of Agg as its own design axis.

Macro — the mission

The Dec-POMDP objective, eq. (2). Interaction enters twice: once shaping each agent's input (through c_{i,t} inside x_{i,t}), and again in how the joint action a_t composes into the next state through T. The research question is how a perturbation confined to the micro level propagates through the bridge and manifests as a change in the macro return — the micro-to-macro gap. Load-bearing assumptions: local agency, partial observability, topology-constrained interaction, a mission-level objective, and trust-by-default among neighbors.

"Twice" — the two channels of interaction

The bridge touches the mission through two mechanically separate channels: an agent's presence affects the outcome once on the way in to others' decisions, and once on the way out through the shared dynamics.

1 · Through the input (information coupling). Before agent i decides, it folds in neighbor messages: c_{i,t} is part of its input x_{i,t} (eq. 3). What others sense and say changes what i believes and chooses. Cut all communication and this channel vanishes. E.g. a neighbor signals which corridor it already swept, so i goes elsewhere.

2 · Through the transition (dynamics coupling). After everyone decides, the next state s_{t+1} = T(·|s_t, a_t) is a function of the joint action, not of i's alone. Actions physically compose: collisions, the union that defines coverage (eq. 4), and the graph that defines connectivity (eq. 5) all depend on where everyone is. This channel is present even with zero communication. E.g. whether i's move newly covers a cell depends on whether anyone else already did.

The two are independent — either can exist without the other. Both are why a micro fault becomes a macro failure, and a compromised agent can inject through either: poisoned messages (channel 1) or corrupting actions (channel 2) — the message/observation vs. action attack surfaces of §7.

02 · mission

The mission: coverage and connectivity

The concrete world is an H×W occupancy grid with obstacles. Two quantities are measured every step and stand in tension.

Coverage (the deliverable) is monotone submodular — over the agents

The right ground set is the agents, not the cells. Over an episode, let C_i ⊆ V be the set of free cells that agent i actually visits (its footprint, determined by the policy). For any subset of agents S ⊆ I, the team's coverage is the size of the union of their footprints:

f(S) = \Big|\, \bigcup_{i \in S} C_i \,\Big|

This f is monotone (adding an agent never reduces coverage) and submodular: for any A ⊆ B ⊆ I and any agent e ∉ B,

f(A \cup \{e\}) - f(A) \;\ge\; f(B \cup \{e\}) - f(B)
(4)

The left side is agent e's marginal coverage on top of the smaller team A; the right side its marginal on top of the larger team B. Because the larger team has already covered more of whatever e would reach, e's fresh contribution shrinks as the baseline grows — diminishing returns. Two agents in the same region is strictly wasteful, so a good policy must partition the space, not pile onto it. (This is the classic maximum-coverage / sensor-placement objective.)

Why this is the crux of the whole paper The difference reward of §4.2 is exactly the discrete derivative of this f: agent i's credit is its marginal coverage f(I) − f(I∖{i}) — the cells only it reached. Submodularity is why a per-agent marginal signal is the natural one and why a shared team reward fails: under a shared number, nothing charges an agent for overlap.

Connectivity (the constraint) via the graph Laplacian

Build the communication graph G_t with an edge between agents within range r. With adjacency matrix A and diagonal degree matrix D, the graph Laplacian is L = D − A, whose eigenvalues order as 0 = λ₁ ≤ λ₂ ≤ … ≤ λ_N. The second-smallest, λ₂, is the algebraic connectivity (Fiedler value) [2]:

G_t \ \text{is connected} \iff \lambda_2\big(\, L(G_t) \,\big) > 0
(5)

with larger λ₂ meaning a more robustly connected team. We use g_conn(s) = −λ₂(L(G_t)) as a differentiable connectivity constraint.

A global quantity in a decentralized world — how we are allowed to use λ₂

λ₂ is a spectral property of the entire graph: computing it needs the full adjacency matrix A — i.e. god's-eye knowledge of every agent's position. Under decentralized, partially observable execution, no agent can compute it. So it is fair to ask how we may use it at all. The answer is that λ₂ plays three distinct roles, and only the first two touch it directly:

1 · As a metric (evaluation). We compute the true λ₂ with god's-eye access purely to measure connectivity when scoring a run. Bookkeeping — the agents never see it.

2 · At training time (this is what CTDE licenses). The centralized critic V_φ(s) and any centrally-computed reward term legitimately see global state — that is the entire point of centralized training (§4.1). A λ₂ term can shape the actors' weights during training without ever being an input to them at run time.

3 · At execution — never the true λ₂. Deployed agents act on local proxies only: local degree (how many neighbors I can still hear), a gossip-propagated estimate of my connected component, or a learned estimate λ̂₂ from the decentralized, size-invariant belief network (§4.3), which infers connectivity from local message passing. The "active λ₂-anchor" relay of §4.5 climbs this estimated λ̂₂, not the god's-eye value. Estimating λ₂ in a decentralized way is itself a distributed message-passing computation (distributed power iteration on L), which is exactly why that belief module is recurrent. And the Fiedler-oracle experiment in §5 — which did hand agents the exact λ₂ — was a deliberate diagnostic whose null result confirms we are not secretly relying on centralized connectivity.

The central tension Coverage (eq. 4) rewards dispersion; connectivity (eq. 5) rewards proximity. Maximizing one naively destroys the other. Every method below attempts max_π J(π) subject to λ₂(G_t) ≥ λ_min for all t.

Agents are labeled blue (nominal) and red (adversarial). Sections 1–6 build and analyze the blue team; Section 7 introduces the adversary formally.

03 · design space

The design space

An "architecture" is not one decision but a point chosen on several independent axes. Each row of the table is one knob; a complete agent design picks exactly one option per row. We hold the others fixed and vary one axis at a time, so any change in the result is attributable to that axis rather than to an uncontrolled confound. Each axis also maps onto a layer of the formalism from §1 — perception and belief are the micro input x_{i,t}; aggregation is the bridge Agg_i; the reward and the connectivity constraint act at the macro level.

04 · methods

Methods, axis by axis

4.1 · Learning method: MAPPO under CTDE, and ES

Policies are neural networks trained primarily with PPO (Proximal Policy Optimization) [3], maximizing a clipped surrogate that bounds each update's policy change:

L^{\mathrm{CLIP}}(\theta) = \mathbb{E}_t\!\left[\, \min\!\big(\, r_t(\theta)\,\hat{A}_t,\ \ \mathrm{clip}\big(r_t(\theta),\, 1-\epsilon,\, 1+\epsilon\big)\,\hat{A}_t \,\big) \,\right]
(6)

with probability ratio r_t(θ) = π_θ(a_{i,t}|o_{i,t}) / π_θ⁻(a_{i,t}|o_{i,t}) (θ⁻ = weights before the update) and advantage Â_t from GAE (generalized advantage estimation) [4]:

\hat{A}_t = \sum_{l \ge 0} (\gamma\lambda)^l\, \delta_{t+l}, \qquad \delta_t = r_t + \gamma\, V(s_{t+1}) - V(s_t)
(7)

The multi-agent instantiation is MAPPO [5] under CTDE: shared-parameter actors π_θ(a_{i,t}|o_{i,t}) execute from local observations, while a centralized critic V_φ(s) sees the global state — only during training, to cut the advantage's variance. At deployment the critic is discarded.

In plain terms — CTDE Train with a coach who sees the whole board (the centralized critic V_φ(s)); deploy players who see only their own patch (the actors π_θ(a_{i,t}|o_{i,t})). We ablated the coach — replacing it with a decentralized per-agent value head — and coverage collapsed to ~8% at scale (a pure huddle), confirming the centralized baseline is load-bearing. (This is also the licence, from §2, to use global λ₂ in the training signal.)

For components where gradients are awkward (a small discrete role-selector) we used evolution strategies (OpenAI-ES) [6], a gradient estimate from Gaussian parameter noise:

\nabla_\theta\, \mathbb{E}_{\epsilon}\!\big[\, F(\theta + \sigma\epsilon) \,\big] = \tfrac{1}{\sigma}\, \mathbb{E}_{\epsilon \sim \mathcal{N}(0,\,I)}\!\big[\, F(\theta + \sigma\epsilon)\, \epsilon \,\big]
(8)

ES trains a disjoint parameter set from the PPO executor and shares only the team return — a stable evolutionary/gradient hybrid. Empirically, fully independent learners (IPPO) matched CTDE at small scale, and both beat a fully centralized joint policy: decentralized execution is competitive, not merely a deployment convenience. The aggregation axis lives inside these networks — the message-fusion operator Agg_i of eq. (3) was swept over sum / mean / max / attention, with max and mean essentially tied and attention no better, at 32×32.

4.2 · Reward decomposition: the difference reward — the lever

The default multi-agent reward is the shared return: every agent gets the team advantage from GAE(R_team). Under a shared signal an agent's gradient is nearly independent of its own position — it is rewarded for team success whether or not it contributed. The submodular structure (§2) then makes the risk-minimizing behavior to stay central, where connectivity is free. Every agent reasons identically and the team converges to a degenerate near-equilibrium: the huddle — λ₂ high, coverage in the single digits.

The fix is the difference reward (Wolpert & Tumer's Wonderful Life Utility) [7], paying each agent its counterfactual marginal contribution:

D_i(z) = G(z) - G(z_{-i})
(9)

where G is the team objective and z_{−i} is the outcome with agent i's effect removed. As noted in §2, for coverage this is precisely the marginal f(I) − f(I∖{i}) — the cells only agent i reached. Because D_i subtracts everything agent i did not influence, it is factored (aligned with the team objective) yet has far higher signal-to-noise for that agent's own decisions — the same insight COMA [8] exploits with a learned counterfactual baseline. We compute D_i exactly from the submodular coverage gain and run per-agent GAE on it, keeping the centralized value as baseline.

In plain terms — difference reward Your bonus = (team score with you) − (team score without you): a leave-one-out credit. Shadowing a teammate onto already-covered cells yields D_i ≈ 0, so the optimum shifts to covering ground no one else does. It converts "don't rock the boat" into "find your own region."

On the 32×32 grid with 10 agents (four terrains, mean of two seeds, evaluated by sampling the stochastic policy — never argmax):

Terrain (32×32, N=10)Shared R_teamDifference D_iΔ
Open4.4%27.7%+23.3
Rooms6.7%18.9%+12.2
Mixed1.7%14.7%+12.9
Crowded4.6%11.7%+7.1

+7 to +23 points, 2–6×. At 16×16/4 the same change lifts a 12% huddle to a 77% sweep.

Key finding The lever is the signal (the term multiplying the log-probability in the policy gradient), not the function approximator. A change to the reward decomposition — not to any network — produced the largest coverage gain in the program. The cost: D_i pays everyone to disperse, so at scale λ₂ falls, re-opening the connectivity problem addressed by roles (§4.5).

4.3 · Perception and belief

Under partial observability each agent maintains a belief — an accumulated estimate of the map and team from local observations (a learned summary of the history h_{i,t}). Findings:

  • Frontier attention. A learned soft attention over compass sectors biases the goal logits toward the most-uncovered direction — a differentiable analogue of Yamauchi frontier exploration [9]. It roughly doubled coverage (13.5% → 31.6% at 32×32, comm_r=5) and was the single most valuable perception change.
  • Occupancy belief. Agents fold sensed cells into a shared three-valued occupancy map (free / wall / unknown), gossiped within each connected component. An early wall-blind bug had confounded prior obstacle results; fixing it was a prerequisite, not an improvement.
  • Size-invariant graph belief (GCRN). A graph-convolutional recurrent network [10] with a per-node GRU and per-step message passing, trained by privileged distillation against the true adjacency. Trained only at 16×16/4, it transferred zero-shot to 32×32/10 at 72–74% adjacency accuracy (vs. ~50% chance). A feedforward "snapshot" estimator hit 88% in-distribution but collapsed out-of-distribution — evidence that graph-state estimation is a recurrent message-passing problem, not single-frame classification. (This is the decentralized λ̂₂ estimator referenced in §2.)
  • Null / negative levers. Neighbor self-attention, extra directional features, and recurrent memory were all approximately neutral. An information-gain bonus reward-hacked — it maximized proximity to unexplored cells rather than entering them; a coverage-scaled bump term dominated it.

4.4 · Action representation: the suspected ceiling

Every agent used a one-step move head, A_i = {N,S,E,W,stay}. We argue this is a representational ceiling. Optimal coverage of a submodular objective is a spatial assignment — "region R_k is agent i's" — but a one-step action cannot commit to or hold a region; it only expresses a local displacement. The untested high-value change is a goal-selection head (choose a frontier node) composed with an A* planner [11] for transit — a change to the action set A_i, not to perception. Independent evidence (region-based exploration policies) corroborates the one-step move as a general limiter.

4.5 · Roles: explorer and active relay

Difference reward disperses the team; roles buy connectivity back. The specialization: explorer (maximize D_i) vs. relay (raise the team's connectivity at a chokepoint). The emergent-vs-authored debate — whether specialization arises from a homogeneous shared policy given the right signal (ROMA [12], RODE [13]) or must be imposed — resolves, in our regime, to a middle path: a hardcoded role assignment failed, but an active relay performing local gradient ascent on its estimated λ̂₂ (§2, role 3 — never the god's-eye value) beat both a homogeneous policy and a static "hold" relay.

A few active relays dominate many static ones and improve both objectives at once. On maze-like "rooms" maps the relay still cannot localize the chokepoint — open work.

4.6 · Connectivity mechanisms, and why per-step enforcement fails

We cast this as a constrained MDP [14] and tried three enforcement families for λ₂(G_t) ≥ λ_min:

  • Hard action mask. Forbid any action whose greedy successor drops λ₂ below λ_min (STAY always feasible). Guarantees connectivity — and is banned here (see callout).
  • Soft penalty. Add −w·max(0, λ_min − λ₂) to the reward. Inert at scale: the huddle already satisfies λ₂ ≫ λ_min, so the penalty is ~0 and never shapes behavior.
  • Lagrangian / PID-Lagrangian. Solve the constrained saddle-point with an adaptive dual variable λ ≥ 0 [15]:
\max_{\pi}\ \min_{\lambda \ge 0}\ \big[\, J(\pi) - \lambda\,\big( c(\pi) - c_0 \big) \,\big], \qquad \lambda \leftarrow \big[\, \lambda + \eta\,\big( c(\pi) - c_0 \big) \,\big]_{+}
(10)

The dual-ascent update needs a violation c(π) − c₀ > 0 to raise λ. But the huddle never violates connectivity — it only fails to cover — so empirically λ stayed flat (≈ 0.30 → 0.30, violation ≈ 0) and the controller was structurally inert; the PID variant smoothed the dynamics but did not move the fixed point. The lesson is precise: a per-step connectivity constraint cannot break a policy whose failure mode is not a connectivity violation.

Why the guaranteed fix is banned The hard mask makes λ₂ ≥ λ_min an invariant — connectivity becomes unbreakable by construction. That is disqualifying for a study of resilience: our object of interest is precisely when and how the constraint is violated, including under a covert adversary (§7). Robustness that is assumed cannot be measured. We keep connectivity a soft, earned property throughout. (This overrides an earlier finding that a hard mask adds ~20 coverage points at small scale.)

A related point from resilient distributed systems: with f Byzantine agents, consensus-style guarantees require (2f+1)-robust graph connectivity (the W-MSR condition) [16] — tying tolerable fault count directly to topology, a bridge to §7.

05 · turning points

Three turning points

The team floods; it does not divide

A fixed-horizon diagnostic measured redundancy — the mean number of agents covering each covered cell. Scaling N=6 → N=18 raised redundancy ~3.7 → ~7.9 while coverage rose only because more area was flooded. Added agents increased overlap, not partition — a direct empirical signature of unsolved submodular credit assignment (eq. 4).

A perfect connectivity signal is a null lever

We supplied agents the exact λ₂ and their per-agent Fiedler-vector component — a god's-eye oracle no deployed agent could compute (the diagnostic promised in §2). Relay formation did not improve (0–2% relays; connectivity no better than baseline). This isolates the bottleneck: not information, but the reward decomposition (nothing paid for relaying) and the action set (a one-step move cannot hold a chokepoint). It falsified a multi-year "richer perception" hypothesis in one controlled experiment.

Reframing the objective

These results moved the question from "which function approximator?" to "what is the per-agent objective, and is the target behavior expressible in the action set?" The difference reward (§4.2) answers the first; the goal-selection head (§4.4) targets the second.

06 · honesty

Measurement & honesty

Reproducibility. Coverage estimates are stable across seeds; connectivity estimates are not — an identical configuration re-ran at 51% vs. 77% connected, a variance that implicates the estimator, not just the policy. We therefore state coverage claims and hold connectivity claims pending deterministic checkpoint evaluation with error bars.

A budget ceiling. With N=10, horizon H=100, and visit-only coverage (cover_r=0), the team makes at most N·H = 1000 visits against ~1024 free cells; a god's-eye Voronoi-partition oracle caps near 72%. So "why not 100%?" is often a budget answer, and we score against the achievable oracle, not total coverage.

Evaluation protocol. Policies are evaluated by sampling π (at ε=0), never by argmax: greedy evaluation destroys the stochastic symmetry-breaking that separates agents, and turned genuine 46% policies into catastrophic 0% ones in ablations.

Methodological stance Reproduce before reporting; know the oracle ceiling; evaluate by sampling; and state which of your own claims you do not yet believe. The connectivity numbers here are explicitly provisional.
07 · resilience

Adversarial resilience: the research core

The nominal blue team is the substrate; the program's target is resilience to covert internal misbehavior. The threat is specified effect-based — by the mission-relevant outcome, not the exploit — as a tuple:

\mathcal{D} = (\, C,\ \alpha,\ \epsilon_s,\ \Delta \,), \qquad k = |C|
(11)
Threat parameters C ⊆ I the compromised set, k = |C| its size · α the per-step action-deviation amplitude · ε_s the local stealth budget · Δ the activation pattern (when the attack is on).

A compromised agent runs an effective policy mixing nominal and adversarial behavior under an activation switch β_{i,t} ∈ {0,1}, with bounded amplitude in total-variation distance:

\hat{\pi}_i = (1-\beta_{i,t})\,\pi_i + \beta_{i,t}\,\tilde{\pi}_i, \qquad d_{\mathrm{TV}}(\tilde{\pi}_i,\, \pi_i) \le \alpha
(12)

The attack is covert if it stays within a per-edge stealth bound on its messages — no single neighbor can reliably distinguish it (by Neyman–Pearson this bounds any local detector's Type-II error):

D_{\mathrm{KL}}\!\big(\, P_{\mathrm{nom}}(m_{i\to j,\,t}) \,\big\|\, P_{\mathrm{pert}}(m_{i\to j,\,t}) \,\big) \le \epsilon_s
(13)

The key phenomenon is propagation: a per-edge-covert perturbation can still shift the whole mission. One-step influence is a total-variation distance between induced next-inputs, and it composes along paths (data-processing inequality + TV triangle inequality) into a multi-hop bound:

\mathrm{Infl}^{\,i \to j}_{\,t:t+\ell} \ \le\ \sum_{\mathrm{paths}}\ \prod_{\mathrm{edges}} \mathrm{Infl}
(14)

The blue/red interaction is a two-player zero-sum POSG (partially observable stochastic game): blue seeks a max-min-robust policy, red the min-cost deviation. The headline quantities are the return gap and the minimum compromise size:

\Delta J(k) = J(\pi,\,\mathrm{clean}) - \min_{\mathcal{D} \in \Phi_k} J(\pi,\,\mathcal{D}), \qquad k^{*}(\theta) = \min\{\, k : \Delta J(k) \ge \theta \,\}
(15)

where Φ_k is the feasible deviation set at compromise size k. Resilience is reported as threat-parameterized trajectory metrics — robustness R_rob = inf_D J(π̂)/J(π), a brittleness index B_brit = sup‖∇Γ‖ (cliff vs. slope), elasticity, and recovery time T_rec. Early results already show a knee at k=1: a single well-placed compromised agent is the most cost-effective attack, because the topology amplifies it (clean 98.5% → 89.6% with one red). Working hypotheses: (H1) comms amplify small compromise at influential graph positions; (H2) adaptive > random attacks at equal budget; (H3) the dominant attack surface — message, observation, or action — is mission-dependent.

Why the substrate had to come first Every quantity above — ΔJ(k), k*, R_rob — is defined relative to a known, competent nominal policy π. Without a blue team that reliably covers and stays connected, there is no baseline against which to measure an attack. Sections 1–6 built that baseline.
08 · open problems

Open problems

Highest-value untested directions, in priority order:

  • Goal-selection + A* action head (§4.4) — a change to A_i to test the representational-ceiling hypothesis.
  • Hierarchical skill selector — a high-level policy over learned options {disperse, regroup, hold}, trained on difference reward; infrastructure built, not yet run at scale.
  • Relay localization on chokepoint maps — a positioning controller that climbs the team's estimated λ̂₂ plus a connectivity-centrality reward.
  • Deterministic connectivity re-measurement (§6) before any connectivity headline.
  • The adversarial POSG of §7, now that the substrate is stable.

The throughline: when a learning system will not produce the target behavior, interrogate the per-agent objective and the action representation before enlarging the model.

Glossary

Dec-POMDP
Decentralized partially observable MDP — the formal model of cooperative agents with local observations and a shared reward (eq. 1–2).
state S
The complete god's-eye configuration (all positions, covered cells, obstacles, comm graph). No agent observes it directly.
observation model O / partial observability
Agent i receives o_{i,t} ~ O(·|s_t,i) — a local, lossy reading of the state. The gap between s_t and o_{i,t} is partial observability.
transition T(s'|s,a)
The environment's dynamics: probability of the next state given the current state and joint action.
x_{i,t}, q_{i,t}
Agent i's per-step input = (observation o_{i,t}, auxiliary vector q_{i,t}). q_{i,t} holds what it receives beyond raw sensing — the aggregated neighbor messages c_{i,t} and its role tag. Belief / occupancy / trust are the derived summary of the history, not raw input.
history h_{i,t} vs. trajectory τ
History = the sequence of one agent's own past inputs x and actions (local, partial); the belief is a learned summary of it. Trajectory = the global sequence of states and joint actions (god's-eye). The agent has only the history.
joint policy π / J(π)
π = (π_1,…,π_N); "joint" because the mission depends on all agents together. J(π) is the expected total reward over trajectories (eq. 2).
Agg_i
The bridge operator (eq. 3): a permutation-invariant, variable-count pooling (sum/mean/max/attention) of neighbor messages into a fixed-size context.
submodularity
Diminishing-returns property of the union-coverage set function over agents (eq. 4). Its marginal is the difference reward.
algebraic connectivity λ₂
Second-smallest eigenvalue of the Laplacian L=D−A. Zero iff disconnected. A global quantity — used only as a metric and in training; execution uses a local estimate λ̂₂ (§2).
PPO / GAE
Proximal Policy Optimization (clipped policy gradient, eq. 6) with Generalized Advantage Estimation (eq. 7).
CTDE / MAPPO
Centralized-training, decentralized-execution; a centralized critic V_φ(s) during training, local actors at deployment.
evolution strategies (ES)
Gradient-free optimization via Gaussian parameter perturbations (eq. 8); used for the discrete role-selector.
difference reward D_i
Counterfactual marginal-contribution signal G(z)−G(z_{−i}) (eq. 9); the discrete derivative of the coverage function.
the huddle
Degenerate equilibrium of a shared return: agents cluster, λ₂ high, coverage near zero.
GCRN
Graph-convolutional recurrent network; a size-invariant belief over the team graph (the decentralized λ̂₂ estimator), trained by privileged distillation.
constrained MDP / Lagrangian
An MDP with a constraint, solved via a saddle-point with an adaptive dual price λ (eq. 10).
threat model D=(C,α,ε_s,Δ)
Effect-based adversary spec: compromised set, action amplitude, local stealth budget, activation pattern (eq. 11–13).
return gap ΔJ(k) / k*
Mission damage from k compromised agents, and the minimum k that forces damage θ (eq. 15).
blue / red
Nominal vs. adversarial agents. Zymera studies covert red agents embedded in a blue team.
